# Environment Example Generator

A specialized agent for generating comprehensive `.env.example` files with documented environment variables, type hints, and proper secrets handling.

## Features

- **Comprehensive Documentation**: Generate .env.example files with detailed comments for each variable
- **Type Hints**: Add type annotations (string, number, boolean, url, path, email, port, secret, array)
- **Secrets Handling**: Properly identify and separate sensitive variables
- **Framework Presets**: Pre-configured variables for common frameworks (Node.js, Python, Go, etc.)
- **Multi-Format Output**: Generate .env, JSON, YAML, or TOML format
- **Validation Hints**: Include regex patterns and allowed values for validation
- **Documentation Comments**: Helpful descriptions for every variable

## Usage

### Direct Agent Invocation

```yaml
agent: env-example-generator-agent
input:
  project_name: my-api
  environment_type: nodejs
  variables:
    - name: NODE_ENV
      description: Application environment
      type: string
      default_value: development
      allowed_values: [development, staging, production]
    - name: DATABASE_URL
      description: PostgreSQL database connection string
      type: url
      required: true
      secret: true
    - name: PORT
      description: Server port number
      type: port
      default_value: "3000"
  framework_presets:
    database: true
    auth: true
    cache: true
  split_secrets: true
```

## Framework Presets

### Database Configuration

Include database-related variables:

```yaml
framework_presets:
  database: true
```

Generates:
- `DATABASE_URL`: Connection string
- `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`: Individual components
- `DATABASE_POOL_SIZE`: Connection pool size

### Cache Configuration

```yaml
framework_presets:
  cache: true
```

Generates:
- `REDIS_URL`: Redis connection string
- `CACHE_TTL`: Cache time-to-live
- `MEMCACHED_*`: Memcached configuration

### Authentication Configuration

```yaml
framework_presets:
  auth: true
```

Generates:
- `JWT_SECRET`: JWT signing secret
- `SESSION_SECRET`: Session encryption key
- `OAUTH_CLIENT_ID`, `OAUTH_CLIENT_SECRET`: OAuth credentials
- `AUTH0_*`: Auth0 configuration

### Storage Configuration

```yaml
framework_presets:
  storage: true
```

Generates:
- `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`: AWS credentials
- `AWS_S3_BUCKET`, `AWS_REGION`: S3 configuration
- `STORAGE_ENDPOINT`: Custom storage endpoint
- `GCS_*`: Google Cloud Storage

### Email Configuration

```yaml
framework_presets:
  email: true
```

Generates:
- `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASSWORD`: SMTP settings
- `SENDGRID_API_KEY`: SendGrid integration
- `MAIL_FROM`: Default sender address

### Monitoring Configuration

```yaml
framework_presets:
  monitoring: true
```

Generates:
- `DATADOG_API_KEY`: Datadog integration
- `SENTRY_DSN`: Sentry error tracking
- `LOG_LEVEL`: Logging verbosity
- `NEWRELIC_LICENSE_KEY`: New Relic APM

### Queue Configuration

```yaml
framework_presets:
  queue: true
```

Generates:
- `RABBITMQ_URL`: RabbitMQ connection
- `AWS_SQS_*`: AWS SQS configuration
- `KAFKA_*`: Apache Kafka settings

## Variable Types

| Type | Description | Example |
|------|-------------|---------|
| string | General text value | `APP_NAME=myapp` |
| number | Numeric value | `PORT=3000` |
| boolean | true/false | `DEBUG=true` |
| url | HTTP/HTTPS URL | `DATABASE_URL=postgresql://...` |
| path | File system path | `STORAGE_PATH=/data` |
| email | Email address | `ADMIN_EMAIL=admin@example.com` |
| port | TCP/UDP port | `PORT=8080` |
| secret | Sensitive data | `API_KEY=` (empty value) |
| array | List of values | `ALLOWED_HOSTS=localhost,127.0.0.1` |

## Input Schema

### Full Input Specification

```yaml
environment_type: nodejs  # Target framework
project_name: my-app      # Project name for comments
split_secrets: true        # Separate secret variables
include_validation: true   # Add validation hints
include_comments: true    # Include descriptive comments
format: env               # Output format: env, json, yaml, toml

variables:
  - name: VARIABLE_NAME
    description: What this variable does
    type: string          # Type hint
    default_value: value # Default value
    required: true        # Is it required?
    secret: false         # Is it sensitive?
    example: sample       # Example value
    validation: "regex:^[a-z]+$"  # Validation pattern
    allowed_values: [dev, staging, prod]  # Enum values

framework_presets:
  database: true
  cache: true
  auth: true
  storage: true
  email: true
  monitoring: true
  queue: true
```

## Best Practices

### Secrets Handling

1. **Always leave secret values empty** in `.env.example`
2. **Use descriptive comments** to indicate expected format
3. **Separate secrets** into a dedicated section
4. **Mark required vs optional** variables clearly

### Variable Naming

- Use `UPPERCASE_WITH_UNDERSCORES`
- Be descriptive but concise
- Use consistent prefixes (`DB_`, `REDIS_`, `AWS_`)

### Documentation

- Include type hints for all variables
- Document required vs optional
- Provide allowed values for enums
- Add validation patterns where applicable
